Bangumi Torrent Finder

一次搜索，聚合全部番剧磁力来源。

聚合 bangumi.moe、acg.rip、動漫花園、nyaa.si 的搜索结果，自动去重并按番剧分组，支持批量复制磁力与打包下载 .torrent 文件。

查看源码 本地启动指南

它能做什么

多源并行检索

四个上游 provider 并发调用，互不阻塞。单个源失败只会作为 warning 返回，不影响整体响应。

智能分组

先跑启发式标题解析得出快速结果，随后后台用 LLM 精炼跨语言（简繁日罗）同作品聚合，首屏不阻塞。

批量打包下载

选中若干条结果后，服务端代理拉取 .torrent 并即时打包为一个 zip 返回，绕过 CORS。

SQLite 搜索缓存

相似查询直接命中缓存，TTL 可配。减少上游压力与 LLM token 消耗。

token 不落前端

所有上游 URL / API key 仅在服务端使用，前端只看到归一化后的字段。

易扩展

新增 provider 只需实现 SearchProvider 接口并在 lib/search.ts 中注册。

快速启动

git clone https://github.com/madeye/bangumi-downloader
cd bangumi-downloader
cp .env.example .env.local
npm install
npm run dev   # http://localhost:3000

常用命令

  • npm run dev启动 Next.js 开发服务
  • npm run build / start生产构建与启动
  • npm run lintESLint（flat config）
  • npm run typechecktsc --noEmit
  • npm testVitest 全量测试

不变量

  • 服务端收口所有上游 token / URL 只在服务端使用，不要从客户端直接 import provider 模块。
  • 容错优先provider 失败通过 warnings 上报，不要让一个坏源拖垮整个聚合响应。
  • 字段契约provider 输出必须符合 SearchResultItem，否则分组与排序都会失效。
